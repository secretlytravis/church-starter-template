Forms

Form elements are minimal and consistent. Labels are uppercase, small, and spaced. Inputs have a 1px border that turns navy on focus. Border-radius is 2px throughout.

Icons

All icons are inline SVGs — no icon font, no external library. They follow a consistent style: stroke-based, no fill, 1.8px stroke-width, rounded caps and joins. Default size is 16×16px in UI contexts, larger where used decoratively.

Icon Style Rules
/* Standard icon — UI context (contact details, meta info) */ <svg viewBox="0 0 24 24" style="width:16px; height:16px;"> stroke="currentColor" /* inherits text color */ fill="none" stroke-width="1.8" stroke-linecap="round" stroke-linejoin="round" </svg> /* Decorative icon — placeholder / empty state */ fill="rgba(200,146,42,0.35)" /* muted gold */ width: 40–72px /* Play button — video thumbnails */ fill="white" filter="drop-shadow(0 2px 8px rgba(0,0,0,0.5))" width: 44–64px

Accessibility

The site aims for WCAG 2.1 AA compliance. Every interactive element must be keyboard accessible, every image must have meaningful alt text, and color contrast must meet minimum ratios.

Color Contrast
White on Navy
✓ Passes AA (10.4:1)
Dark on Cream
✓ Passes AA (14.1:1)
Gold on Navy
⚠ Use for labels only (4.3:1)
ARIA & Semantic Rules
/* Navigation */ <nav aria-label="Main navigation"> <ul role="list"> <button aria-label="Open menu" aria-expanded="false/true"> /* Images */ <img alt="Pastor First Last"> /* meaningful alt */ aria-hidden="true" /* decorative SVGs */ /* Sections */ <section aria-labelledby="section-heading-id"> /* Forms */ <label for="input-id"> /* always explicit for= */ role="status" /* success/error messages */ /* Links */ target="_blank" rel="noopener" /* all external links */ aria-label="YouTube channel" /* icon-only links */

✓ Accessibility Do Use semantic HTML elements (nav, main, section, footer). Give every form input an explicit label. Add aria-label to icon-only buttons and links. Use aria-labelledby to connect sections to their headings. Test keyboard navigation — every interactive element must be reachable by Tab.
✗ Accessibility Don't Use color alone to convey information. Use placeholder text as a substitute for labels. Add decorative images without aria-hidden="true". Create click targets smaller than 44×44px. Remove focus outlines without providing an alternative focus indicator.

Voice & Tone

The writing across the site should feel like Your Pastor speaking — warm, honest, direct, and doctrinally serious without being academic. Never preachy. Never corporate. Never vague.

✓ Do Write in plain, honest sentences. Use the em dash for natural pauses. Be direct about what the church believes. Speak warmly to first-time visitors. Use "we" for the church. Let Scripture do the heavy lifting. End invitations with genuine warmth, not pressure.
✗ Don't Use church jargon without explanation. Be vague about doctrine to avoid offending. Use corporate language ("community," "journey," "space"). Over-explain or pad with filler sentences. Make visitors feel guilty for not coming. Use exclamation points excessively.

Tone by Context
Context Tone Example
Homepage / Hero Welcoming, clear, unhurried "Come as you are — we mean that."
Romans Road Honest, plain, serious, hopeful "This isn't meant to shame us. It's meant to be honest."
New Believer Pastoral, warm, practical "You were not saved to figure this out on your own."
Beliefs Confident, precise, grounded "We believe in one sovereign God, eternally existing as Father, Son, and Holy Spirit."
Contact / Prayer Gentle, no pressure, personal "We believe God hears and answers prayer. Whatever you are carrying right now — we would be honored to carry it with you."

Usage Rules

A few hard rules that keep the site consistent as it grows.

✓ Always Use Playfair Display for all headings and display text. Use gold as an accent — borders, labels, dividers, and CTAs. Keep section padding at 60–80px. Link all scripture references to ESV.org. Keep the max content width at 960px. Use the gold 2px divider after every section title.
✗ Never Use gold as a background color for large areas. Use more than two typefaces. Add a new color outside the defined palette. Use border-radius larger than 4px. Stack two navy sections back to back without a light section between. Add inline styles that override the CSS variables.

CSS Variables Reference
:root { /* Brand colors */ --navy: #082846; /* Primary dark — nav, hero, CTAs */ --navy-mid: #0d3b66; /* Secondary dark — hover states */ --gold: #c8922a; /* Accent — borders, labels, buttons */ --gold-light: #e8b45a; /* Hover state for gold */ /* Backgrounds */ --cream: #f9f5ee; /* Default page background */ --white: #ffffff; /* Cards, forms, even sections */ /* Text */ --text-dark: #1a1a1a; /* Headings on light backgrounds */ --text-mid: #4a4a4a; /* Body copy */ }
